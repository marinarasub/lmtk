\end{DoxyItemize}\hypertarget{index_autotoc_md23}{}\doxysubsection{Compiling / Building}\label{index_autotoc_md23}
You can build using CMake and using vcpkg as a dependency manager. Currently only vcpkg is supported for CMake.

First download \href{https://cmake.org/}{\texttt{ CMake}} and \href{https://vcpkg.io/en/index.html}{\texttt{ vcpkg}} if you haven\textquotesingle{}t already.

Make sure to put vcpkg in your environment variables.

Steps to build\+:


\begin{DoxyItemize}
\item Run {\ttfamily vcpkg install} on the top level folder (lmtk/). This should download required dependencies.
\item Use CMake at the top level folder (lmtk/).
\begin{DoxyItemize}
\item If CMake can\textquotesingle{}t find vcpkg toolchain, try specifying VCPKG\+\_\+\+ROOT in CMake environment varables
\item Select {\ttfamily BUILD\+\_\+\+TEST} if you want to build tests
\end{DoxyItemize}

\end{DoxyItemize}\hypertarget{index_autotoc_md32}{}\doxysubsubsection{Testing}\label{index_autotoc_md32}
I am using \href{https://github.com/google/googletest}{\texttt{ Google Test}} to unit test, which should be installed using vcpkg.

Utility funcitons are tested \char`\"{}normally\char`\"{} using assertions and such.

Due to the nature of the project, a lot of tests require file read/write which is difficult to make portable. If you decide to run tests, expect a lot of failed tests unless you supply for your own files. In addition, many features are quite complex and undergo minor changes often, so the some of the tests are not set in stone.

In the future, I will add test media files to the repository.
